Getting a Next.js app live with a custom domain is a surprisingly satisfying process — and it's free for personal projects. Here's exactly how to do it.

## The stack

- **Next.js** — the framework
- **GitHub** — source control and CI trigger
- **Vercel** — hosting and deployment
- **Cloudflare** — DNS management (and optionally, a CDN/WAF layer)

## Step 1: Create the Next.js app

```bash
npx create-next-app@latest my-app --typescript --tailwind --app
cd my-app
```

## Step 2: Push to GitHub

```bash
gh repo create my-app --public --source . --remote origin --push
```

## Step 3: Deploy to Vercel

```bash
npm install -g vercel
vercel login
vercel --prod
```

Vercel auto-detects Next.js and sets up the build pipeline. It also connects to your GitHub repo so every push triggers a new deployment.

## Step 4: Connect your domain

In Vercel, go to **Settings → Domains** and add your domain. Vercel will show you the DNS records to add.

### In Cloudflare

Add a **CNAME** record:

```
Type:    CNAME
Name:    www
Target:  <value from Vercel>
Proxy:   OFF (grey cloud)
```

> Keep the proxy **off** so Vercel can manage SSL via Let's Encrypt.

### Domain verification

If your domain was previously linked to another Vercel account, add a TXT record:

```
Type:    TXT
Name:    _vercel
Value:   vc-domain-verify=...
```

Once DNS propagates (usually under 5 minutes with Cloudflare), your domain goes green in Vercel.

## That's it

Your site is now live at your custom domain, auto-deploying on every push to `main`. No servers to manage, no certificates to renew.
